Hypertext Markup Language

Hypertext Markup Language is the language used to markup the structure and content of documents(i.e. webpages) in the WWW

It was created by Tim Berners-Lee

    Aspects:

  • structure and content
  • aesthetics/presentation - well layed-out
  • behavioral - client-side (JavaScript)

Versions and History:

  • HTML 1.0 - first version of HTML
  • HTML 2.0 - embodied in a technical document (RFC in 1866) and was the first standard HTML which was published in November 1995.
  • HTML 3.0 - W3C Recommendation
  • HTML 3.2
  • HTML 4.0
  • HTML 4.01 - W3C Recommendation (December 24 1999)
  • Web Hypertext Technology Group (WHATWG) - Ian Hickson started this group and develop the next version of HTML since XHTML is not that great because of the strict syntax.
  • HTML 5.0 (2014) - supports HTML flexible syntax and XHTML strict syntax
  • HTML 5.1 (2016)

Typical Start of HTML

Extensible Hypertext Markup Language

Extensible Hypertext Markup Language is the reformulation of HTML to XML

It was developed by the World Wide Web Consortium (W3C) to help web developers to make transition from HTML to XML

XHTML is almost identical to HTML 4.01

The rules are very strict (case sensitive)

  • strict
    • - deprecate presentational aspects.
  • transitional
    • - you can still use deprecated presentation aspect.
  • frameset
    • - to partition the browser into multiple frames.

Content Model

  • Heading
    • - the header of a section.
    • h1, h2, h3, h4, h5, h6
  • Sectioning
    • - content that defines the scope of the headings and footers.
    • article, aside, nav, section
  • Embedded
    • - content that imports another resource into the document.
    • audio, canvas, embed, iframe, img, math, object, svg, video
  • Phrasing
    • - the text of the document aswell as the elements.
    • a, abbr, area, b, bdi, bdo, br, button, canvas, cite, code, command, dfn, em, embed, i, iframe, input, ins, kbd, label, map, mark, math, object, output, progress, q, ruby, s, samp, script, select, small, span , strong, sub, sup, svg, textarea, time, u, var, wbr
  • Flow
    • - elements that are mostly used in the body of the document.
    • a, abbr, address, area, article, aside, audio, b, bdi, bdo, blockquote, br, button, canvas, cite, code, command, datalist, del, details, dfn, div, dl, em, embed, fieldset, figure, footer, form, header, hgroup, hr, i , iframe, img, input, ins, kbd, keygen, label, map, mark, math, menu, meter, nav, noscript, object, ol, output, p, pre, progress, q, ruby, s, samp, script, section, select, small, span, strong, style, sub, sup, svg, table, textarea, time, u, ul, var, video, wbr
  • Interactive
    • - content that is intended for user interaction.
    • a, audio, button, details, embed, iframe, img, input, keygen, label, menu, object, select, textarea, video
  • Metadata
    • - content that sets up the presentation of the content with other documents.
    • base, command, link, meta, noscript, script, style, title
